# 效果与伤害结算规则

## 核心原则

1. **先特效后伤害**：一个 hit 的所有特效在伤害计算之前处理
2. **深度优先级联**：每施加一个效果，立即检查是否有监听者需要插入结算
3. **"xx后"延迟触发**：描述中含"后"字的效果（`deferred: true`），在当前 hit 完整结算后才执行
4. **同帧多效果同时处理**：同一技能描述中的多个效果同时入队

## 完整结算流程

```
施放技能 (ACTION_START)
  → [监听施放技能的 buff 触发]
  → 法术附着
    → [监听附着的 buff 触发]
    → 附着累积达到阈值 → 法术爆发（伤害，不清空附着）
  → 法术异常
    → [监听异常的 buff 触发]
    → 异常效果（碎甲、导电debuff等）
  → 物理异常
    → [监听物理异常的 buff 触发]
    → 物理异常效果（破防层、碎冰等）
    → 异常伤害（猛击、碎冰等）
  → 其他特效（消耗附着、施加脆弱等）
    → [监听特效的 buff 触发]
  → 技能伤害 + 装备 proc 伤害
  → [默认 trigger：不影响本次 hit 的 buff 施加]
技能完成
  → [deferred trigger："xx后"效果执行]
```

## 实现机制

### 帧内栈式执行 (SimulationEngine)
- 同帧事件深度优先：handler 产生的级联事件立即排在未处理事件之前
- 入队顺序保证"先特效后伤害"：附着 → 异常 → 物理异常 → 伤害

### 帧快照 (frameSnapshot)
- 帧开始时快照敌人状态（附着层数、破防层数等）
- 消耗类 handler 从快照读取倍率（如别礼连携 hit2 从快照读寒冷附着层数）

### deferred 触发
- `EffectTrigger.deferred = true`：收集但不立即执行
- 帧末 `flushDeferred()`：所有主事件处理完后执行

### 法术爆发
- 同元素附着达到 4 层 → 爆发伤害
- 爆发**不清空**附着层数（仅物理异常和法术异常清空附着）

## "xx后" / deferred 的正确理解

`deferred: true` 的含义不是"帧末执行"，而是"**触发它的动作/子动作完整结算完毕后**"。

例：点剑套监听物理异常 → 猛击触发物理异常 → 猛击的整个生命周期（消耗破防→碎晶反应→猛击伤害→碎晶伤害→结晶消耗→天赋触发）全部完成后 → 点剑套才结算。

这对应 SimulationEngine 中 `processEventWithCascade` 的 `popAndFlushDeferred()`：
deferred triggers 在事件的整个级联子树完成后才执行。

## 已验证案例

| 案例 | 结论 |
|------|------|
| 佩丽卡战技 | 附着在伤害前施加 |
| 佩丽卡连携技 | 法术异常在伤害前施加 |
| 管理员战技（猛击） | 物理异常在伤害前处理 |
| 管理员连携+战技（宏愿） | 载体buff→武器trigger→伤害，消耗链完整级联 |
| 埃特拉连携（冻结目标） | 击飞+物理脆弱同时处理，所有伤害（击飞/碎冰/技能）均吃到脆弱 |
| 别礼天赋1"消耗附着后" | deferred 触发：hit 完整结算后才施加寒冷脆弱 |
| 管理员战技+显赫声名+点剑套 | 猛击cascade: 猛击/碎晶吃到现实静滞+显赫声名; deferred后: 点剑/战技吃到本质瓦解+显赫声名, 不吃现实静滞 |

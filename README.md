基于 [end-axis](https://github.com/floating-sky/end-axis) 二次开发，核心模拟引擎与大部分前端逻辑已全面重写。（原作者联系方式没写，也可能是我智力有问题找不到，所以没有授权）

## 功能

- **排轴编辑器** — 双模式（自由摆放 / 拟真模拟），支持技能放置、连携技队列、普攻连段
- **V2 伤害模拟引擎** — 逐 hit 结算，11 乘区伤害公式，buff/debuff/异常/失衡完整模拟 （这个是真的，而且结算顺序大概是做了验证的，基本都没问题）（但是话又说回来了，伤害计算功能我还没做完）
- **技能变体自动识别** — （草了我写不出这么专业的描述，总之就是放技能的时候会根据实际情况来结算，比如有附着会转法术爆发/法术异常这样，技能被打断也会有对应效果）
- **多方案管理** — 最多 10 个方案快速切换对比（然而实际上我的屏幕超过5个方案就会被其他的按钮盖住导致选不到对应方案（挠头））
- **国际化** — 中文（默认）/ English （其实我也不知道到底支不支持英文，假装支持吧，反正看得懂这段的应该是不刚需英文的，看不懂的我说你不看用户协议就打勾用不了是你的问题（？））

## 本地运行 （我不知道能不能运行，建议拿docker或者虚拟机，不然把你电脑搞坏了不要骂我qq）

```bash
cd apps/endaxis-web
npm install          # Node ^20.19.0 || >=22.12.0
npm run dev          # http://localhost:1420
npm test             # Vitest 测试
npm run type-check   # vue-tsc 类型检查
```

## 技术栈（我看不懂）

- **前端**: Vue 3 + Vite + Pinia + Element Plus
- **模拟引擎**: 纯 TypeScript，无后端依赖
- **编译器**: 自研时间轴编译系统（时间冻结、效果解析、打断检测）
- **测试**: Vitest
